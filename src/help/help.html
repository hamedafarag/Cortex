Cortex AI Review Assistant Settings

Everything Cortex can do

An AI review copilot that lives inside the GitHub PR — highlight code, review the whole change, and post line-anchored comments, all on the Claude subscription or API key you already pay for. Your code never touches a third-party server.

Ask about the code

Highlight & ask

Select any code in the diff and ask a question. The answer streams back in the dock, grounded in the authoritative diff hunk and the PR's title + description — so the model judges the change against its stated intent, not just local correctness.

Keep asking in the same thread — each follow-up carries the prior turns.

Review the whole PR

Whole-PR review with severity tags

One click (no selection needed) reviews every changed file and streams a findings list. Each finding leads with a severity so you can triage blocker-vs-nit at a glance, and promotes to a real review comment via Use as comment.

Blocker Major Minor Nit Praise
Specialist lenses

Focus the review on one dimension

Pick a lens beside Review to scope the pass to Security, Performance, Error handling, or Readability — the model reports only findings in that lens, and skips the rest.

Test-gap check

Spot changed code that lacks tests

A fast, no-AI pass over the changed-file list flags which source files changed without a matching test change — instant, free, and offline. An approximation by file name, not coverage (and it says so).

Summarize the PR

Orient before you dive in

A whole-PR summary — TL;DR, key changes, a per-file gloss, and a 1–5 review-effort rating — grounded in the actual diffs, so you know what you're walking into before reading a line.

Suggest a fix

One-click committable suggestions

Ask for a fix on the selected lines and get a GitHub suggestion block the author can apply in one click. Multi-line selections anchor to the whole range.

Faster, clearer comments

Canned snippets & Conventional Comments

Drop in out-of-the-box review snippets, or prepend a Conventional Comments label (suggestion, issue, nit, … · blocking / non-blocking / if-minor) so authors instantly see what's blocking vs. optional. Then Post to line sends it as a line-anchored review comment.

Post with a safety net

Confirm before, undo after

Posting is the one real public write, so Cortex guards it both ways. Post to line first shows a confirm bar with the exact repo · path:line — nothing is written until you confirm. After a successful post, a 10-second Undo deletes the comment in one click, and a Refresh shows it inline in the diff. Cortex never posts on its own.

And the foundations

Two interchangeable backends

Your Anthropic API key or your Claude subscription (via the Claude Code CLI native host), behind one interface with automatic fallback.

Native, adaptive UI

The dock inherits GitHub's light/dark Primer theme so it feels built-in — and every status pairs colour with an icon + label (color-blind safe).

Local & private

Keys and token live only in your browser. Code goes only to Anthropic (your account) and api.github.com (your token). No third-party SaaS — and obvious secrets are masked before anything is sent.

Remembers per PR

Your conversation and the comment you're drafting are saved per PR and restored when you come back — after a reload or navigating away. Stored only in your browser.
