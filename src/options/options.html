GitHub — posting comments

Lets the dock post line-anchored review comments to your PRs, as you. Sent only to api.github.com (no third party).
Recommended: a fine-grained token (smallest blast radius). When creating it:
  1. Resource owner → the organization that owns the repos — not your personal account
  2. Repository access → Only select repositories → the repos you review
  3. Permissions → Pull requests: Read and write — nothing else
Org gate: for org repos the org must allow fine-grained tokens and may need an owner to approve the token (it stays pending until then). If posting returns 404, that's the cause. A classic token with the repo scope also works (for SSO orgs, Configure SSO → Authorize) but can touch every repo you can access. Revoke anytime in Developer settings.
